Audio Console v4.1 (Recommended)
Console Adapter Script

Path to the audio console adapter. This is loaded as the "wake" module in Akari.

Wake Word Detection

How Akari detects the wake word. openWakeWord is recommended (local, on-device, ONNX models).

openWakeWord Model URL

URL to a .onnx keyword model. The keyword name is auto-extracted from the filename.

Speech Recognition (ASR)

How Akari transcribes your speech after the wake word.

Transformers Model

Which ASR model to download and run in-browser. Smaller = faster but less accurate.

whisper.cpp Server URL

Base URL of your whisper-server (llamafile whisperfile). Example: http://127.0.0.1:8080

Web Speech Language

BCP-47 language code for browser SpeechRecognition.

Wake Sensitivity

How confident Akari must be before triggering. Lower = more sensitive (more false triggers). Higher = stricter.

Voice Activity Detection

How sensitive the silence/speech detector is. Lower = picks up quieter speech. Higher = needs louder speech.

Text Wake Words

Comma-separated words to strip from transcripts. Example: "hey akari, akari, computer". If no sound wake is used, these words trigger recognition.

Privacy
Require wake word before listening

When ON, Akari only transcribes speech after a wake word (sound or manual). When OFF, all speech is transcribed and text wake words are checked in the transcript.

XL Audio Cache (Retroactive Recording)

Keeps a rolling window of mic audio in RAM. You can retrieve it after the fact (playback, transcribe, or save). Must be set before the console starts.

Enable XL Cache

Cache duration:

In milliseconds. Range: 1000 (1 second) to 3600000 (1 hour). At 16kHz mono, 1 minute ≈ 1.9MB, 1 hour ≈ 115MB.

Only store detected speech (saves RAM)
